# Система доставки

**Система доставки** действует как мост между [Виртуальным портфелем](/ru/features/holdings) и физическим инвентарем. Она позволяет игрокам конвертировать свои виртуальные активы в реальные предметы.

## Концепция

Если у игрока есть 100 "Долей Железа" и ему нужно железо для строительства, он может запросить "доставку".

1. Игрок выбирает актив для вывода.
2. Доли удаляются из виртуального портфеля.
3. Соответствующие физические предметы добавляются в инвентарь.

## Команды

```bash
/endex deliver <предмет> <количество>
```

Или нажмите на иконку доставки (обычно вагонетка или сундук) в меню `/holdings`.

## Комиссия за доставку

Чтобы сбалансировать экономику и управлять виртуальной торговлей против физической, вы можете настроить **Комиссию за доставку**.

```yaml
delivery:
  enabled: true
  fee-per-item: 0.0 # Фиксированная стоимость за предмет
  fee-percentage: 0.02 # 2% от текущей стоимости предмета
```

Если комиссия установлена, игрок должен заплатить эту сумму, чтобы конвертировать свои акции в предметы.

## Ограничения

- **Инвентарь полон**: Если инвентарь игрока полон, доставка будет отменена или выполнена частично (в зависимости от конфигурации).
- **Недоставляемые предметы**: Вы можете определить определенные предметы как "чисто виртуальные" (недоставляемые), установив `deliverable: false` в `items.yml`. Это полезно для "Акций компании" или рыночных индексов, которые не существуют физически.
