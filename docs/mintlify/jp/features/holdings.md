# ホールディングスシステム（保有資産）

**Holdings**（ホールディングス/ポートフォリオ）システムにより、プレイヤーはアイテムを物理的にインベントリに保管することなく投資することができます。これは現実世界で株式や先物契約を購入するのと似ています。

## 仕組み

物理的な「ダイヤモンド」を購入する代わりに、プレイヤーは「ダイヤモンドの持分」を購入します。

1. プレイヤーは現在の市場価格を支払います。
2. 持分は仮想ポートフォリオに保存されます。
3. 物理的なアイテムはプレイヤーに**渡されません**。
4. 後で、プレイヤーは持分を新しい市場価格で売却できます。

### 利点

- **インベントリ管理不要**: プレイヤーはチェストを必要とせずに数千単位を取引できます。
- **純粋な投機**: 資源を集めずにトレーダーを演じたいプレイヤーに最適です。
- **保護**: 保有資産は盗まれたり、燃えたり、死亡によって失われたりすることはありません。

## ポートフォリオコマンド

- `/holdings`: 投資を確認するためのポートフォリオGUIを開きます。
- `/endex invest <アイテム> <数量>`: 保有資産を購入します。
- `/endex divest <アイテム> <数量>`: 保有資産を売却します。

## 設定

`config.yml` でこのシステムを有効または無効にできます：

```yaml
holdings:
  enabled: true
  max-holdings-per-item: 10000 # プレイヤーごとの制限
  tax-rate: 0.05 # 保有資産の利益に対する5%の税金
```

## キャピタルゲイン税

投機によって経済が急速に膨張するのを防ぐために、保有資産の売却に対して特定の税金（`tax-rate`）を課すことができます。この税金は、保有資産を売却する際に利益または総額から差し引かれます。
