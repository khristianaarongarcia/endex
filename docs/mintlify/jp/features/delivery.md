# 配送システム

**配送システム**は、[仮想ポートフォリオ](/jp/features/holdings)と物理インベントリの間の架け橋となります。これにより、プレイヤーは仮想の保有資産を実際のアイテムに変換できます。

## コンセプト

プレイヤーが100個の「鉄の持分」を所有していて、建築のために鉄が必要な場合、「配送」をリクエストできます。

1. プレイヤーは引き出す保有資産を選択します。
2. 保有資産は仮想ポートフォリオから削除されます。
3. 対応する物理アイテムがインベントリに追加されます。

## コマンド

```bash
/endex deliver <アイテム> <数量>
```

または、`/holdings` メニューから配送アイコン（通常はトロッコやチェスト）をクリックします。

## 配送手数料

経済のバランスを取り、仮想取引と物理取引を管理するために、**配送手数料**を設定できます。

```yaml
delivery:
  enabled: true
  fee-per-item: 0.0 # アイテムごとの固定コスト
  fee-percentage: 0.02 # アイテムの現在価値の2%
```

手数料が設定されている場合、プレイヤーは株式をアイテムに変換するためにこの金額を支払う必要があります。

## 制限

- **インベントリ満杯**: プレイヤーのインベントリが満杯の場合、配送はキャンセルされるか、部分的にのみ行われます（設定によります）。
- **配送不可アイテム**: `items.yml` で `deliverable: false` を設定することで、特定のアイテムを「純粋に仮想」（配送不可）として設定できます。これは、物理的に存在しない「企業株」や市場指数に役立ちます。
