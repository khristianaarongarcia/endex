# 배송 시스템

**배송 시스템**은 [가상 포트폴리오](/ko/features/holdings)와 물리적 인벤토리 사이의 다리 역할을 합니다. 이를 통해 플레이어는 가상 보유 자산을 실제 아이템으로 전환할 수 있습니다.

## 개념

플레이어가 100개의 "철 지분"을 소유하고 있고 건축을 위해 철이 필요한 경우 "배송"을 요청할 수 있습니다.

1. 플레이어는 인출할 보유 자산을 선택합니다.
2. 보유 자산은 가상 포트폴리오에서 제거됩니다.
3. 해당 물리적 아이템이 인벤토리에 추가됩니다.

## 명령어

```bash
/endex deliver <아이템> <수량>
```

또는 `/holdings` 메뉴에서 배송 아이콘(일반적으로 광산 수레나 상자)을 클릭합니다.

## 배송 수수료

경제의 균형을 맞추고 가상 거래와 물리적 거래를 관리하기 위해 **배송 수수료**를 설정할 수 있습니다.

```yaml
delivery:
  enabled: true
  fee-per-item: 0.0 # 아이템당 고정 비용
  fee-percentage: 0.02 # 아이템 현재 가치의 2%
```

수수료가 설정된 경우 플레이어는 주식을 아이템으로 전환하기 위해 이 금액을 지불해야 합니다.

## 제한 사항

- **인벤토리 가득 참**: 플레이어의 인벤토리가 가득 찬 경우 배송이 취소되거나 부분적으로만 수행됩니다(설정에 따라 다름).
- **배송 불가 아이템**: `items.yml`에서 `deliverable: false`를 설정하여 특정 아이템을 "순수 가상"(배송 불가)으로 설정할 수 있습니다. 이는 물리적으로 존재하지 않는 "회사 주식"이나 시장 지수에 유용합니다.
