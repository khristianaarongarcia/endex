# 동적 가격 시스템

The Endex의 핵심은 수요와 공급에 실시간으로 반응하는 동적 가격 알고리즘입니다.

## 개념

가격이 고정(정적)된 기존 상점과 달리 The Endex는 가격을 자동으로 조정합니다:

- **플레이어가 구매하면** 재고가 (이론적으로) 감소하고 수요가 증가합니다 -> **가격이 상승합니다**.
- **플레이어가 판매하면** 재고가 증가하고 공급이 증가합니다 -> **가격이 하락합니다**.

이를 통해 플레이어가 싸게 사서 비싸게 팔아 이익을 얻을 수 있는 활기찬 경제가 만들어집니다.

## 가격 알고리즘

가격은 다음 요소를 고려한 공식을 사용하여 계산됩니다:

1. **기본 가격**: 관리자가 설정한 시작점.
2. **변동성 (Volatility)**: 가격이 얼마나 빨리 변하는지.
3. **거래량**: 얼마나 많은 아이템이 거래되고 있는지.
4. **제한**: 극심한 인플레이션/디플레이션을 방지하기 위한 최소 및 최대 가격.

### 단순화된 공식

$$
P_{new} = P_{current} \times (1 + \frac{Volume \times Volatility}{Constant})
$$

## 변동성 설정

각 아이템은 `items.yml`에서 고유한 변동성을 가질 수 있습니다.

- **낮은 변동성 (0.1 - 0.3)**: 안정적인 가격, 천천히 변함. 기본 자원(돌, 나무)에 적합합니다.
- **중간 변동성 (0.4 - 0.7)**: 일반적인 변동. 광석이나 몹 드롭에 적합합니다.
- **높은 변동성 (0.8 - 1.5)**: 매우 불안정한 가격, 고위험 고수익. 희귀 아이템에 적합합니다.

```yaml
GOLD_INGOT:
  base-price: 50.0
  volatility: 0.5  # 중간 변동성
```

## 가격 제한

경제를 보호하기 위해 엄격한 제한을 설정할 수 있습니다:

- `min-price`: 가격이 이 값 아래로 떨어지지 않습니다.
- `max-price`: 가격이 이 값을 초과하지 않습니다.

```yaml
DIAMOND:
  base-price: 100.0
  min-price: 20.0   # 20 아래로 떨어지지 않음
  max-price: 500.0  # 500을 초과하지 않음
```

## 가격 재생

활동이 없을 때 시간이 지남에 따라 가격이 천천히 기본 가격으로 돌아오도록 설정할 수 있습니다. 이는 가격이 극단적인 상태에 무기한 머무르는 것을 방지합니다.

`config.yml`의 `price-regeneration`에서 이를 활성화하세요.
